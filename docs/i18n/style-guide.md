# Стиль перевода EN → RU

<sub>Last reviewed: 2026-06-10 (i18n typo fix)</sub>

Этот документ — источник истины по стилю, тону и правилам при переводе `docs/content/**/*.md` на русский. Используется вместе с [`glossary.ru.yml`](./glossary.ru.yml) и [`prompt.md`](./prompt.md).

## Целевая аудитория

Русскоязычный разработчик, **средний уровень**, работает с JavaScript/TypeScript и REST API. Знает базовые технические термины на английском. Пишем не «c нуля для журналиста» — можно оставлять в виде выражения вроде «payload», «callback», «throttle».

## Тон

- **Формальный, нейтральный.** Никакого «дружище» или «давайте хакнем». Но и не перегруженный канцелярит в стиле «регламента».
- **Обращение «вы»** — с маленькой буквы. Пример: «вы можете выбрать любой из трёх вариантов».
- **Безличная форма** где уместно: «Метод возвращает» вместо «Метод вы получите».
- **Настоящее время** для инструкций: «Создаёте вебхук», не «Нужно будет создать».
- Избегать буквальных переводов-калек («обратный вызов» вместо «callback»).

## Что ПЕРЕВОДИМ

### Проза

Весь видимый текст вне кода:

- Абзацы и списки.
- **Текст заголовков** (но НЕ якорь `{#anchor}` — см. ниже).
- **Текстовая часть ссылок:** `[текст](url)` — «текст» переводим, URL НЕ трогаем.
- **Текст внутри MDC-компонентов:** `::warning\nТекст внутри\n::` — текст переводим, название компонента и `::` НЕ трогаем.
- **Подписи таблиц** и содержимое ячеек (кроме тех, где тех. идентификаторы).
- **`title="..."`** внутри MDC-компонентов, **`label="..."`** в `tabs-item`, **`alt="..."`** в картинках.

### Frontmatter (YAML в верху файла)

Переводим **значения**, ровно ключи НЕ трогаем:

```yaml
title: "Английское название"          # → «Русское название»
description: "English description."   # → «Русское описание.»
navigation.title: "Short"             # → «Коротко»
links:
  - label: "GitHub source"            # → «Исходник на GitHub»
    iconName: GitHubIcon              # НЕ трогаем (enum)
    to: https://github.com/...        # НЕ трогаем (URL)
```

## Что НЕ ПЕРЕВОДИМ — ни одного символа

### Код-блоки и inline-код

````markdown
```ts
// всё внутри этого блока — НЕ трогаем
import { B24Hook } from '@bitrix24/b24jssdk'
```

`inline-код` в тексте — НЕ трогаем.
````

**Исключение:** комментарии в коде (`// comment` или `/* ... */`), которые являются частью педагогического примера, **можно** перевести (`// Get current user` → `// Получаем текущего пользователя`). Строки в `console.log(...)` и `logger.error(...)` — ОСТАВЛЯЕМ английскими для предсказуемости фильтров логов.

### Якоря заголовков

```markdown
## Инициализация {#initialization}
```

Якорь `{#initialization}` — НЕ переводить. Служит для стабильных URL-ссылок (при изменении текста заголовка внешние ссылки не ломаются).

**Ссылки на якоря — одна решётка.** Внутристраничная ссылка на заголовок пишется с **одной** `#`: `[текст](#anchor)`. Двойная решётка `[текст](##anchor)` — баг (ссылка не работает). В EN-оригинале иногда встречается `##` — не копируйте дословно, исправляйте на одинарную и заводите issue в upstream (пример — [bitrix24/b24jssdk#131](https://github.com/bitrix24/b24jssdk/issues/131)). Проверяйте все `](#...)` после перевода.

**Технически важно.** Синтаксис `## Текст {#anchor}` в `@nuxt/content` v3 **не работает из коробки** — встроенный парсер воспринимает `{#...}` как обычный текст и выводит его прямо в DOM, а `id` собирает из всего текста, включая фигурные скобки. Получается дубль типа `id="add-to-a-vue-project-add-to-a-vue-project"`, а в заголовке видно `{#add-to-a-vue-project}`.

Решение реализовано в `docs/nuxt.config.ts` через плагин [`remark-custom-heading-id`](https://www.npmjs.com/package/remark-custom-heading-id):

```ts
content: {
  build: {
    markdown: {
      remarkPlugins: {
        'remark-custom-heading-id': {}
      }
    }
  }
}
```

Плагин — runtime-зависимость, объявлена в **корневом** `package.json` (`dependencies.remark-custom-heading-id`). Если при отладке заголовки выглядят сломанными — проверьте: `pnpm install` запущен из корня, плагин подключён в `nuxt.config.ts`. Подробнее — `AGENTS.md` → «Кастомные якоря заголовков».

### Имена классов, методов, npm-пакетов

Сохраняем оригинал:

- `B24Hook`, `B24Frame`, `B24OAuth`, `LoggerFactory`, `AjaxResult`, `SdkError`.
- `callMethod`, `fetchListMethod`, `batchByChunk`, `prepareCode`.
- `@bitrix24/b24jssdk`, `@bitrix24/b24jssdk-nuxt`, `@bitrix24/b24ui-nuxt`, `@nuxt/content`, `@bitrix24/b24icons-vue/...`.
- Названия иконок: `Bitrix24Icon`, `B24Button`, `PlayLIcon`.

### URL, ID, data-атрибуты, CSS-классы

`href="..."`, `id="..."`, `class="..."`, `data-*="..."`, `[link](https://...)` URL'ы — НЕ трогаем.

### Frontmatter служебные поля

Сохраняем оригинал:

- `category` (enum: `actions`, `tools`, `logger`, ...)
- `restApiVersion` (enum: `rest-api-ver2`, `rest-api-ver3`)
- `badge` (если это короткий лейбл вроде `beta`, `new`, `deprecated` — сохраняем английский)
- `audited` (YYYY-MM-DD)
- `links.iconName` (имя иконки)
- `links.to` (URL)
- `links.target` (`_blank` или `_self`)
- `navigation.shadow`, `links.color`, `links.size` и пр. enum-ы

### Атрибуты доступности с названием продукта

`aria-label="Bitrix24 REST API"`, `alt="Bitrix24 UI"`, `title="Bitrix24 webhook"` — в таких контекстах НЕ переводим бренд/продукт. Текст рядом (напр. `aria-label="Скопировать"`) — переводим.

## Типографика и пунктуация

- **Длинное тире**: `—` (не `-` и не `–`). Пример: «Метод — это способ...».
- **Кавычки**: «ёлочки» первый уровень, „лапки" внутри. Пример: «Параметр „order" отвечает за...».
- **Кавычки в код-блоках** НЕ трогаем (остаются `"..."` или `'...'`).
- **Неразрывный пробел** («40 мс», «1 МБ») — обычный пробел ОК (markdown не сломается).
- **Единицы** НЕ переводим: `ms`, `KB`, `MB`, `GB`, `px`, `em`, `rem`, `vh`, `vw`. Пишем как в оригинале («задержка 100 ms» или «задержка 100 мс» — и так, и так ОК, выбирайте по контексту).
- **Пунктуация в списках**: если пункт — полное предложение, в конце точка. Если короткая фраза (одно слово/словосочетание) — без точки. Единообразно в рамках одного списка.

## Бренд vs правообладатель

Главное правило — см. [AGENTS.md](../../AGENTS.md):

- **Бренд платформы**: `Bitrix24` (в тексте, `aria-label`, npm-пакетах, именах классов).
- **Правообладатель** (LICENSE, footer копирайт, frontmatter `author`): `Bitrix` (в юр. тексте) / `Битрикс` (в русском UI). Без цифры.

## MDC-разметка — конспект

`@nuxt/content` использует MDC — расширение markdown с компонентами. Синтакс НЕ трогаем:

```mdc
::warning
Текст предупреждения — ПЕРЕВОДИМ.
::

::note{color="info"}
Текст примечания — ПЕРЕВОДИМ.
::

::tabs
  :::tabs-item{label="Сервер"}      ← label ПЕРЕВОДИМ
  Текст — ПЕРЕВОДИМ.
  :::
::

::steps{level="3"}
### Шаг первый     ← заголовок ПЕРЕВОДИМ
Текст.
::

::code-group{sync="pm"}
\`\`\`bash [pnpm]                 ← не трогаем
pnpm add ...
\`\`\`
::
```

## Частые ловушки

1. **Перевод якоря** — `## Инициализация {#инициализация}`. НЕПРАВИЛЬНО. Оставьте `{#initialization}`.
2. **Перевод имени свойства** в прозе — напр. «в параметре „время"». НЕПРАВИЛЬНО — «в параметре „timeout"».
3. **Изменение ключей frontmatter** — напр. `заголовок: ...` вместо `title: ...`. НЕПРАВИЛЬНО.
4. **Перевод комментария внутри компонента**: `::code-group{sync="pm"}` → `::code-group{sync="пм"}`. НЕПРАВИЛЬНО — это параметр-enum.
5. **«Получить» вместо «получают»** в заголовках/списках — не варьируйте вид, будьте консистентны.
6. **Удаление подключения `remark-custom-heading-id`** в `nuxt.config.ts` — без него все `{#anchor}` сломают заголовки в браузере. НЕ удалять при правках конфига.
7. **Двойная решётка в ссылке на якорь** — `[текст](##anchor)`. НЕПРАВИЛЬНО — одна решётка `[текст](#anchor)`. Иногда приходит из EN-оригинала — не копируйте баг дословно, исправляйте и заводите issue в upstream.

## Проверка перевода

Перед коммитом каждого файла:

1. `pnpm run docs:dev` — страница рендерится без ошибок.
2. Код-блоки не изменены (поравнить с EN-оригиналом).
3. MDC-синтакс целиком (без лишних пробелов, с закрывающими `::`).
4. Навигация в боковом меню показывает русский `navigation.title`.
5. Якоря в ToC боковой панели — английские (slug из `{#en-anchor}`), фрагмент `{#...}` НЕ виден в тексте заголовка (если виден — подключение `remark-custom-heading-id` сломано).
6. Ссылки на якоря — одна решётка `](#...)`, не `](##...)`. Проверьте: `grep -nE '\]\(##' <файл>` — должно быть пусто.
