# AI промпт для перевода

Скопируйте блок ниже в Cursor/Claude Code/Windsurf и передайте AI вместе с файлом (или прямо конкретный markdown) — получите RU-перевод с учётом конвенций проекта.

## Базовый промпт (для одного файла)

````
Переведи markdown-файл с английского на русский по правилам проекта.

Контекст проекта:
- Это документация SDK @bitrix24/b24jssdk для русскоязычных разработчиков.
- Обязательные источники (прочитай их ПЕРЕД переводом):
  - docs/i18n/glossary.ru.yml  — словарь терминов; обращай внимание на поля `keep_en` и `note`
  - docs/i18n/style-guide.md   — стиль, тон, типографика, MDC-разметка
  - AGENTS.md                  — общие правила проекта (брендинг, ссылки, sync)
- Сайт русскоязычный (RU-only), без переключателя языков.

Переведи:
- Всю прозу вне код-блоков
- Текст заголовков (НО НЕ якорь `{#en-anchor}` — оставь как есть)
- Frontmatter: title, description, navigation.title, badge (если это обычный текст, а не enum)
- Содержимое MDC-компонентов:
  - `::warning`, `::note`, `::tabs`, `::steps`, `::code-group` и их вложенных `:::tabs-item`, `:::caution`
  - Переводим только текст внутри. Синтакс компонентов (`::`, `:::`, имя, `{ параметры }`) НЕ трогай.
  - Атрибуты label="...", title="...", alt="..." (видимые в UI надписи) — ПЕРЕВОДИМ.
  - sync="pm", color="info", level="3" и прочие enum/служебные — НЕ трогай.

НЕ ПЕРЕВОДИ:
- Код-блоки ``` и inline `backticks` — ни единого символа
- Имена классов/методов/функций SDK: B24Hook, B24Frame, B24OAuth, callMethod, fetchListMethod, AjaxResult, SdkError и т.п.
- Между hook и B24Hook — важная разница:
  - «hook» в прозе как синоним webhook → «вебхук» («set up a hook» → «настроите вебхук»).
  - Имя класса `B24Hook` — НИКОГДА не трогай (ни «Вебхук-клиент», ни «B24Хук», ничего).
- npm-пакеты: @bitrix24/b24jssdk, @bitrix24/b24jssdk-nuxt, @bitrix24/b24ui-nuxt, @nuxt/content, @bitrix24/b24icons-vue/...
- URL’ы: в ссылках [text](url) переводи text, но НЕ url
- Якорьные идентификаторы `{#en-anchor}` — оставить английскими
- Frontmatter ключи (title:, description:, links: и т.д.) — не переименовывать
- Frontmatter служебные поля: category, restApiVersion, links.iconName, links.to, links.target, audited — оставить
- Специфические термины Bitrix24, не имеющие русских эквивалентов:
  - placement — не «размещение» и не «позиция»
  - scope, callback, payload, throttle, debounce, backoff, retry-on-error — см. glossary keep_en
  - composable (термин Vue/Nuxt) — «composable» или «composable-функция»
- aria-label с названием продукта ("Bitrix24 REST API", "Bitrix24 UI on Telegram") — НЕ переводить
- CSS-классы, ID, data-атрибуты
- Строки в console.log / logger.*() — оставить английскими (для предсказуемости фильтров лога)

Тон: формальный, нейтральный. Обращение «вы» с маленькой. Безличная форма где уместно.

Типографика:
- Длинное тире —
- Кавычки «ёлочки» первый уровень, „лапки“ внутри
- Используй термины из glossary.ru.yml — если там помечено keep_en: true, оставь английский вариант

Проверьте перед возвратом:
1. Кол-во открывающих ``` = кол-во закрывающих ``` (баланс code-fence блоков).
2. Кол-во строк `:::` до и после равны (вложенные MDC-компоненты).
3. Якоря `{#...}` в результате ровно такие же как в исходнике.
4. Frontmatter YAML парсится (отступы сохранены).
5. Ни одно имя класса/метода из SDK не переведено.
````

## Компактный вариант (для пакетного перевода по разделу)

Если переводите сразу папку (напр. `docs/content/docs/1.getting-started/`) в Cursor Composer или через Claude Code Agent:

````
Persona: опытный технический транслейтор док-сайта SDK.
Project: bitrix-tools/b24jssdk (RU-only зеркало bitrix24/b24jssdk).

Следуй правилам из этих файлов ПЕРЕД первым переводом:
- docs/i18n/glossary.ru.yml  (обрати внимание на `keep_en: true` и `note`)
- docs/i18n/style-guide.md
- AGENTS.md («Брендинг и копирайт» и «Язык UI-строк»)

Задача: переведи все .md в этой папке на русский инплейс (сохраняя файлы на месте).

Ограничения (критичны):
- Код, MDC-синтакс, якоря {#...}, frontmatter-ключи, npm-пакеты, URL’ы — не трогать.
- frontmatter поля category/restApiVersion/links.iconName/links.to/audited — оставь как есть.
- console.log и logger.*() в код-примерах — английские.
- aria-label с названием продукта ("Bitrix24 REST API") — НЕ переводить.
- `:::tabs-item` и другие вложенные MDC-блоки — тройные двоеточия НЕ менять на двойные.
- Параметры MDC-компонентов в фигурных скобках (`{sync="pm"}`, `{level="3"}`, `{color="info"}`) — НЕ переводить.

Специфика терминов:
- `hook` в прозе («set up a hook») → «вебхук».
- `B24Hook` как имя класса — НЕ трогать.
- `callback` самостоятельный («pass a callback») → «callback». «error handler» → «обработчик ошибок».
- `placement` — оставьте английский («при обработке placement»).
- `composable` — «composable» или «composable-функция» (термин Vue/Nuxt).
- При первом входе нового термина — «русское пояснение (`english_term`)»; дальше одним из вариантов.

Типографика: длинное тире —, кавычки « » снаружи, „ “ внутри.

Проверьте перед результатом: баланс ```/```, баланс `:::`/`:::`, якоря `{#...}` неизменны, frontmatter ключи не переведены, имена классов SDK (B24Hook, B24Frame, B24OAuth) НЕ переведены.

Результат по каждому файлу: полный переведённый markdown без изменения структуры и без побочных комментариев от себя.
````

## После AI-прохода (ручная корректура)

1. Пробежите глазами весь файл. Ищите:
   - Английские фразы, которые AI «забыл» перевести.
   - Лишние кальки «обратный вызов», «полезная нагрузка» — верните по словарю.
   - Переведённые имена SDK (`Клиент B24` вместо `B24Hook`).
   - Сломанный MDC: лишние пробелы в `::warning`, переведённый `label=`, `:::` превращённые в `::`.
2. `pnpm run docs:dev` — откройте страницу, визуально сверьте с оригиналом.
3. Коммит по файлу/разделу с conventional-сообщением: `feat(stage-3b): translate getting-started/installation/vue`.
