## Обзор {#overview}

SDK для работы с REST API Bitrix24 предоставляет набор инструментов для взаимодействия с API Bitrix24 из JavaScript/TypeScript-приложений.

::rest-api-version-only
#rest-api-ver2
Библиотека поддерживает как REST API версии 2 (процесс устаревания начат, но всё ещё широко используется), так и новую версию 3 (процесс внедрения начат).
::

## Архитектура SDK {#sdk-architecture}

SDK построен на трёх основных классах, которые наследуются от общего базового класса и предоставляют единый интерфейс для работы с REST API:

### Три класса для разных сценариев использования {#three-classes-for-different-usage-scenarios}

| Класс | Назначение | Где использовать | Особенности |
|-------|---------|--------------|----------|
| `B24Frame`{lang="ts-type"} | Работа из приложений, встроенных в интерфейс Bitrix24 | На стороне клиента (браузер) | Автоматически доступен в iframe Bitrix24-приложения, использует токен текущего пользователя |
| `B24Hook`{lang="ts-type"} | Работа из автономных серверных приложений через вебхук | На стороне сервера | Использует постоянный токен вебхука, **нельзя использовать в браузере** |
| `B24OAuth`{lang="ts-type"} | Работа из автономных серверных приложений через OAuth 2.0 | На стороне сервера | Использует OAuth-токен с возможностью обновления, **нельзя использовать в браузере** |

### Общий интерфейс {#common-interface}

Все три класса наследуются от общего базового класса и предоставляют одинаковый набор методов:

Все три класса реализуют интерфейс [`TypeB24`](https://github.com/bitrix24/b24jssdk/blob/main/packages/jssdk/src/types/b24.ts), который определяет единый набор операций:

```ts-type
type TypeB24 = {
  readonly isInit: boolean
  init(): Promise<void>
  destroy(): void

  getLogger(): LoggerInterface
  setLogger(logger: LoggerInterface): void

  get auth(): AuthActions

  // REST API (v2 / v3) — call, callList, fetchList, batch, batchByChunk
  get actions(): ActionsManager

  // Helper utilities (healthCheck, ping)
  get tools(): ToolsManager

  setRestrictionManagerParams(params: RestrictionParams): Promise<void>

  getTargetOrigin(): string
  getTargetOriginWithPath(): Map<ApiVersion, string>

  getHttpClient(version: ApiVersion): TypeHttp
  setHttpClient(version: ApiVersion, client: TypeHttp): void
}
```

REST-методы доступны через `actions.v2.*` и `actions.v3.*`:

```ts-type
// $b24 — экземпляр B24Frame | B24Hook | B24OAuth

await $b24.actions.v2.call.make(options)         // Promise<AjaxResult>
await $b24.actions.v2.callList.make(options)     // Promise<Result<T[]>>
$b24.actions.v2.fetchList.make(options)          // AsyncGenerator<T[]>
await $b24.actions.v2.batch.make(options)        // Promise<CallBatchResult>
await $b24.actions.v2.batchByChunk.make(options) // Promise<Result<T[]>>

// v3 симметрична
await $b24.actions.v3.call.make(options)
// ...

await $b24.tools.healthCheck.make({ requestId })  // Promise<boolean>
await $b24.tools.ping.make({ requestId })         // Promise<number>
```

## Ключевые возможности {#key-features}

### Основные методы API {#main-api-methods}

- **`Call`** — базовый метод для вызова любых REST API-методов
- **`CallList`** — автоматическое получение всех данных из list-методов с пагинацией
- **`FetchList`** — поэтапная обработка больших объёмов данных через AsyncGenerator
- **`Batch`** — выполнение до 50 команд в одном запросе
- **`BatchByChunk`** — автоматическое разбиение больших наборов команд на батчи

### Вспомогательные инструменты {#helper-tools}

- **`HealthCheck`** — проверка доступности REST API
- **`Ping`** — измерение времени отклика API

## Выбор подходящего класса {#choosing-the-right-class}

### Для клиентских приложений (браузер) :badge{label="client-side" class="align-text-top"} {#for-client-applications-browser-badge}

Используйте `B24Frame` — он доступен в [iframe Bitrix24-приложений](/docs/getting-started/installation/vue/#init):

::rest-api-version-only
#rest-api-ver2
```ts
// В приложении, встроенном в Bitrix24
async () => {
  $b24 = await initializeB24Frame()
  const response = await $b24.actions.v2.call.make({
    method: 'crm.contact.get',
    params: { id: 123 }
  })
}
```

#rest-api-ver3
```ts
// В приложении, встроенном в Bitrix24
async () => {
  $b24 = await initializeB24Frame()
  const response = await $b24.actions.v3.call.make({
    method: 'tasks.task.get',
    params: { id: 123 }
  })
}
```
::

### Для серверных приложений с постоянным доступом :badge{label="server-side" class="align-text-top"} {#for-server-applications-with-permanent-access-badge}

Используйте `B24Hook` с вебхуком:

::rest-api-version-only
#rest-api-ver2
```ts
import { B24Hook } from '@bitrix24/b24jssdk'

const $b24 = B24Hook.fromWebhookUrl('https://your_domain.bitrix24.com/rest/1/webhook_code/')

const response = await $b24.actions.v2.call.make({
  method: 'crm.contact.get',
  params: { id: 123 },
  requestId: 'server-request'
})
```

#rest-api-ver3
```ts
import { B24Hook } from '@bitrix24/b24jssdk'

const $b24 = B24Hook.fromWebhookUrl('https://your_domain.bitrix24.com/rest/1/webhook_code/')

const response = await $b24.actions.v3.call.make({
  method: 'tasks.task.get',
  params: { id: 123 },
  requestId: 'server-request'
})
```
::

## Общие методы (доступны во всех классах) {#common-methods-available-in-all-classes}

### Архитектура методов {#method-architecture}

| Метод | Возвращаемое значение | Память | Использование | Подходит для |
|-------|----------------------|--------|---------------|-------------------|
| `Call`{lang="ts-type"} | `Promise<AjaxResult>`{lang="ts-type"} | Минимум | Одиночные вызовы | Получение отдельных элементов, вызов методов без пагинации |
| `CallList`{lang="ts-type"} | `Promise<Result<T[]>>`{lang="ts-type"} | Все данные в памяти | `await response.getData()` | Малые и средние объёмы данных, когда все данные нужны сразу |
| `FetchList`{lang="ts-type"} | `AsyncGenerator<T[]>`{lang="ts-type"} | Обработка по частям | `for await (const chunk of generator)` | Большие объёмы данных, потоковая обработка, ограниченная память |
| `Batch`{lang="ts-type"} | `Promise<CallBatchResult>`{lang="ts-type"} | Зависит от объёма | Пакетные операции до 50 команд | Массовые операции, оптимизация сетевых запросов |

::rest-api-version-only
#rest-api-ver2

> **Автоматические предупреждения**
>
> SDK автоматически отслеживает использование методов, доступных в REST API v3:
>
> ```json
> "The method {method_name} is available in restApi:v3. It's worth migrating to the new API."
> ```
>
> Это позволяет постепенно мигрировать с устаревающего API v2 на современный v3.

::

## Рекомендации по выбору метода {#method-selection-recommendations}

### Для получения данных {#for-data-retrieval}

1. **Отдельные элементы** → используйте `Call`
2. **Малые списки (до 1000 записей)** → используйте `CallList`
3. **Большие списки (тысячи записей)** → используйте `FetchList`
4. **Несколько отдельных элементов** → используйте `Batch`

### Для изменения данных {#for-data-modification}

1. **Одиночные операции** → используйте `Call`
2. **Массовые операции (до 50)** → используйте `Batch`
3. **Массовые операции (более 50)** → используйте `BatchByChunk`

### Для отладки и мониторинга {#for-debugging-and-monitoring}

1. **Проверка доступности API** → используйте `HealthCheck`
2. **Измерение скорости отклика** → используйте `Ping`

## Лучшие практики {#best-practices}

### Обработка ошибок {#error-handling}

Всегда проверяйте `isSuccess` и обрабатывайте ошибки:

::rest-api-version-only
#rest-api-ver2
// @check-ignore: top-level return in error-handling illustration
```typescript
const response = await $b24.actions.v2.call.make({
  method: 'some.method',
  params: { /* параметры */ },
  requestId: 'unique-id'
})

if (!response.isSuccess) {
  console.error('Error:', response.getErrorMessages().join('; '))
  // Дополнительная логика обработки ошибок
  return
}

// Работа с успешным результатом
const data = response.getData()
```

#rest-api-ver3

const response = await $b24.actions.v3.call.make({

  // Дополнительная логика обработки ошибок

### Управление запросами {#request-management}

- `requestId`{lang="ts-type"} — всегда указывайте уникальный идентификатор запроса для отслеживания и дедупликации
- **Логирование** — используйте встроенные механизмы логирования для отладки
- **Таймауты** — настраивайте таймауты для длительных операций

## Миграция с REST API v2 на v3 {#migration-from-rest-api-v2-to-v3}

Bitrix24 постепенно переходит на REST API версии 3. Рекомендуется:

1. **Следить за предупреждениями** от SDK о доступности методов в v3
2. **Использовать** [документацию REST API v3](https://apidocs.bitrix24.ru/api-reference/rest-v3/index.html)
3. **Планировать миграцию** существующих проектов
