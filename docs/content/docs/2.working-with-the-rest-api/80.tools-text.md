`Text`{lang="ts-type"} — это синглтон (`new TextManager()`), экспортируемый из `@bitrix24/b24jssdk`. Для операций с датами использует [Luxon](https://moment.github.io/luxon/).

```ts
import { Text } from '@bitrix24/b24jssdk'

console.log(Text.getUuidRfc4122())          // '019323ac-8ace-725b-a3dc-6a7c333da066'
console.log(Text.getDateForLog())           // '2026-05-04 09:53:51'
console.log(Text.numberFormat(1234.567, 2)) // '1,234.57'
```

## Идентификаторы {#identifiers}

| Метод | Описание |
|----|----|
| `getRandom(length = 8)` | Случайная строка `[a-z0-9]` заданной длины. |
| `getUniqId()` | UUID v4 (по возможности, генерируется локально). |
| `getUuidRfc4122()` | UUID v7 (упорядоченный по времени). Используется SDK как request id по умолчанию. |

## Кодирование сущностей {#entity-encoding}

| Метод | Описание |
|----|----|
| `encode(value)` | Заменяет `&`, `<`, `>`, `'`, `"` их entity-кодами (без завершающей `;`). |
| `decode(value)` | Обратно `encode` — также обрабатывает числовые сущности (`&#38`, `&#60`, …). |

::note
`encode` / `decode` намеренно опускают завершающую `;` (например, `&amp` вместо `&amp;`). Это соответствует поведению legacy Bitrix Framework.
::

## Преобразование типов {#type-conversion}

```ts-type
toNumber(value: any): number          // 0.0 при неудаче парсинга
toInteger(value: any): number         // toNumber(parseInt(value, 10))
toBoolean(value: any, trueValues?: string[]): boolean // 'true'/'y'/'1'/1/true → true
```

## Хелперы регистра {#case-helpers}

```ts-type
toCamelCase(str: string): string
toPascalCase(str: string): string
toKebabCase(str: string): string
capitalize(str: string): string
```

`toCamelCase` распознаёт `-`, `_` и пробелы как разделители. `toPascalCase` — это `capitalize(toCamelCase(str))`. `toKebabCase` разбивает по границам заглавных букв (`getUserId` → `get-user-id`).

## Форматирование чисел {#number-formatting}

```ts-type
numberFormat(
  value: number,
  decimals?: number,        // по умолчанию 0
  decPoint?: string,        // по умолчанию '.'
  thousandsSep?: string     // по умолчанию ','
): string
```

Тот же алгоритм, что Bitrix24 использует на сервере — нечисловые значения становятся `0`, дробная часть округляется, разделитель тысяч вставляется каждые три цифры слева от десятичной точки.

## Хелперы дат (Luxon) {#date-helpers-luxon}

```ts-type
toDateTime(dateString: string, template?: string, opts?: DateTimeOptions): DateTime
toB24Format(date: string | Date | DateTime): string
getDateForLog(): string  // 'yyyy-MM-dd HH:mm:ss'
```

`toDateTime` использует `DateTime.fromFormat`, когда передан `template`, иначе `DateTime.fromISO`. `toB24Format` выводит формат, который Bitrix24 ожидает в REST-payload: `yyyy-MM-dd'T'HH:mm:ssZZ`.

## Query-строка {#query-string}

```ts-type
buildQueryString(params: any): string
```

Строит строку `application/x-www-form-urlencoded` из обычного объекта. Значения-массивы становятся парами `key[i]=value`.
