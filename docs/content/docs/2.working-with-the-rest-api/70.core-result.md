`Result<T>`{lang="ts-type"} — это единый тип возврата SDK для хелперов высокого уровня (`callBatch`, `B24HelperManager.loadData`, `OptionsManager.save`, …). Класс реализует `IResult<T>`{lang="ts-type"}; типизированные REST-хелперы (`call.make`, `callList.make`, …) возвращают его специализированный подкласс [`AjaxResult<T>`](/docs/working-with-the-rest-api/core-ajax-result/).

## Использование {#usage}

```ts
import { Result } from '@bitrix24/b24jssdk'

function parseConfig(raw: string): Result<{ theme: string }> {
  try {
    const data = JSON.parse(raw)
    if (typeof data?.theme !== 'string') {
      return Result.fail<{ theme: string }>('theme is required', 'config:theme')
    }
    return Result.ok({ theme: data.theme })
  } catch (error) {
    return Result.fail<{ theme: string }>(error as Error, 'config:json')
  }
}

function applyConfig() {
  const rawJson = '{"theme":"light"}'
  const result = parseConfig(rawJson)
  if (!result.isSuccess) {
    console.error(result.getErrorMessages())
    return
  }
  const { theme } = result.getData()!
  console.log('theme:', theme)
}
applyConfig()
```

## Конструктор {#constructor}

```ts-type
new Result<T>(data?: T)
```

`data` по умолчанию `null`{lang="ts-type"}.

## Статические фабрики {#static-factories}

```ts-type
Result.ok<U>(data?: U): Result<U>
Result.fail<U>(error: Error | string, key?: string): Result<U>
```

## Геттеры {#getters}

| Геттер | Тип | Описание |
|----|----|----|
| `isSuccess` | `boolean`{lang="ts-type"} | `true`, когда карта ошибок пуста. |
| `errors` | `Map<string, Error>`{lang="ts-type"} | Внутреннее хранилище ошибок (по соглашению — только для чтения). |

## Методы данных {#data-methods}

```ts-type
setData(data: T | null | undefined): Result<T>
getData(): T | null | undefined
```

`setData` поддерживает цепочки вызовов. Примечание: `AjaxResult`{lang="ts-type"} **отключает** `setData` и выбрасывает `ReferenceError`{lang="ts-type"} при вызове.

## Методы ошибок {#error-methods}

```ts-type
addError(error: Error | string, key?: string): Result<T>
addErrors(errors: (Error | string)[]): Result<T>
hasError(key: string): boolean
getErrors(): IterableIterator<Error>
getErrorMessages(): string[]
```

Когда `key` опущен, `addError` генерирует UUID v7 через [`Text.getUuidRfc4122()`](/docs/working-with-the-rest-api/tools-text/), чтобы дублирующиеся строки всё равно получали отдельные слоты.

## `toString` {#tostring}

```ts-type
toString(): string
```

Красиво печатает результат для логов:

- успех → `Result(success): {<data JSON>}`
- неудача → `Result(failure): {<data JSON>}\nErrors: <messages joined by ", ">`

Экземпляры `Error` внутри данных сериализуются как `{ name, message, stack }`. При неудаче сериализации возвращается `'[Unable to serialize data]'`.
