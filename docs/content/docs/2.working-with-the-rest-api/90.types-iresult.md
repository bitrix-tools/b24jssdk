`IResult<T>`{lang="ts-type"} отражает паттерн «флаг успеха + данные + набор ошибок», используемый по всему SDK. Конкретные классы:

- [`Result<T>`{lang="ts-type"}](/docs/working-with-the-rest-api/core-result/) — обобщённый.
- [`AjaxResult<T>`{lang="ts-type"}](/docs/working-with-the-rest-api/core-ajax-result/) — REST-ориентированный (неизменяемый, постраничный, декодирует конверты ошибок Bitrix24).

## Форма {#shape}

```ts-type
interface IResult<T = any> {
  readonly isSuccess: boolean
  readonly errors: Map<string, Error>

  setData(data: T): IResult<T>
  getData(): T | null | undefined

  addError(error: Error | string, key?: string): IResult
  addErrors(errors: (Error | string)[]): IResult
  getErrors(): IterableIterator<Error>
  getErrorMessages(): string[]
  hasError(key: string): boolean

  toString(): string
}
```

`AjaxResult<T>`{lang="ts-type"} сужает `setData` до `never` (неизменяемый ответ) и переопределяет `getData()`, чтобы возвращать [`SuccessPayload<T>`{lang="ts-type"}](https://github.com/bitrix24/b24jssdk/blob/main/packages/jssdk/src/types/payloads.ts) при `isSuccess`.

## Типичный паттерн использования {#typical-usage-pattern}

```ts
import type { IResult } from '@bitrix24/b24jssdk'

function consume<T>(result: IResult<T>) {
  if (!result.isSuccess) {
    for (const message of result.getErrorMessages()) {
      console.error(message)
    }
    return
  }
  const data = result.getData()
  // ...
}
```
