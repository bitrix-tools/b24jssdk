`useFormatter()`{lang="ts-type"} возвращает два общих форматтера:

- `formatterNumber` — синглтон `FormatterNumbers`, форматирование чисел с учётом локали.
- `formatterIban` — синглтон `FormatterIban`, предзагруженный **всеми** спецификациями стран IBAN (и региональными французскими subdivisions: GF, GP, MQ, RE, PF, TF, YT, NC, BL, MF, PM, WF).

```ts
import { useFormatter } from '@bitrix24/b24jssdk'

const { formatterNumber, formatterIban } = useFormatter()

console.log(formatterIban.printFormat('GB29NWBK60161331926819'))
// 'GB29 NWBK 6016 1331 9268 19'

console.log(formatterIban.isValid('DE89370400440532013000'))
// true
```

::note
`useFormatter()` регистрирует спецификации IBAN при каждом вызове, но нижележащий `FormatterIban` — синглтон, поэтому добавление спецификации для того же кода страны ничего не делает. Многократный вызов `useFormatter()` дёшев.
::

## `FormatterNumbers` {#formatternumbers}

Обёртка над `Intl.NumberFormat` с учётом локали (валюта, десятичные, проценты). Экземпляр общий для всего пакета; если вы замените его локаль, изменение увидят все потребители.

## `FormatterIban` {#formatteriban}

Валидирует и красиво печатает IBAN для поддерживаемых стран. Ключевые методы:

- `isValid(iban: string): boolean` — проверка контрольной суммы + длины.
- `printFormat(iban: string, sep?: string): string` — группы по четыре символа.
- `electronicFormat(iban: string): string` — убирает пробелы, переводит в верхний регистр.
- `addSpecification(spec: IbanSpecification): void` — зарегистрировать нестандартную страну.
